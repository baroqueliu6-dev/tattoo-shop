## 📋 事件经过

### 初始状态（2026-03-18）
| 项目 | 名称 | 用途 | 状态 |
|------|------|------|------|
| GitHub 仓库 | `tattoo-shop` | 代码仓库 | ✅ 正确 |
| Vercel 项目 | `tattoo-site` | 部署项目 | ❌ 名称不一致 |
| 域名 | `13801380.site` | 主域名 | 已配置到 `tattoo-site` |

### 错误操作（2026-03-19）
1. **推送代码到错误仓库**
   - 本地 Git remote 配置了 `origin` → `tattoo-shop`（正确）
   - 但 Vercel 连接的是 `tattoo-site`（不存在或不同步）
   - 导致 Vercel 没有自动部署新代码

2. **删除项目**
   - 删除了 Vercel 项目 `tattoo-site`
   - **后果:** 域名 `13801380.site` 的配置丢失！

3. **发现剩余项目**
   - `tatoo-shop-2024`（拼写错误，少一个 t）
   - `tatoo-shop`（拼写错误，少一个 t）
   - 这两个项目虽然连接了正确的 GitHub 仓库，但**没有配置域名**

## 🔍 根本原因分析

### 1. 命名不一致
- **GitHub 仓库:** `tattoo-shop`（正确拼写）
- **Vercel 项目:** `tattoo-site`（错误拼写）
- **问题:** 没有在项目创建时统一命名规范

### 2. 缺少关键信息记录
- 没有记录域名绑定在哪个 Vercel 项目上
- 没有记录 DNS 配置详情
- 依赖记忆而非文档

### 3. 操作前没有确认
- 删除项目前没有检查域名绑定状态
- 没有确认删除后的影响范围
- 冲动操作，没有复盘习惯

### 4. Git Remote 配置混乱
- 本地有多个 remote（`origin` 和 `tattoo-site`）
- 没有明确哪个是主要的推送目标
- 推送后没有验证 Vercel 是否自动部署

## 💡 经验教训

### 1. 命名规范（最重要！）
```
✅ 正确做法：
- GitHub 仓库：tattoo-shop
- Vercel 项目：tattoo-shop（与 GitHub 保持一致）
- 本地目录：tattoo-shop

❌ 错误做法：
- GitHub: tattoo-shop
- Vercel: tattoo-site（不一致！）
- Vercel: tatoo-shop（拼写错误！）
```

### 2. 关键信息必须记录
**项目配置文件应该包含：**
```markdown
## 项目信息
- GitHub: https://github.com/baroqueliu6-dev/tattoo-shop
- Vercel: https://vercel.com/baroqueliu6-devs-projects/tattoo-shop
- 主域名：13801380.site
- 子域名：tattoo.13801380.site

## DNS 配置
- A 记录：@ → 76.76.21.21
- CNAME: www → cname.vercel-dns.com
- CNAME: tattoo → cname.vercel-dns.com
```

### 3. 删除操作前的检查清单
- [ ] 这个项目绑定了哪些域名？
- [ ] 删除后域名会怎样？
- [ ] 有没有备份配置？
- [ ] 能否先解绑域名再删除？
- [ ] 是否通知了相关人员？

### 4. Git 操作规范
```bash
# 查看 remote 配置
git remote -v

# 确认推送到正确的仓库
git push origin main

# 验证 Vercel 是否自动部署
# 访问 Vercel Dashboard 检查部署状态
```

### 5. 定期备份配置
- 每月检查一次域名配置
- 记录 DNS 设置变更
- 保存 Vercel 项目配置截图
